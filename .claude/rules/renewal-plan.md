# 관리자단 리뉴얼 기획서

> 작성일: 2026-03-22
> 대상: `/public_html/sysop/` 전체
> 방향: 정보 밀도는 유지하면서 현대적인 SaaS 어드민 UX로 개선

---

## 1. 현황 분석

### 1-1. 규모
| 항목 | 현황 |
|------|------|
| 주요 모듈 수 | 57개 |
| JSP 파일 수 | 721개 |
| 네비게이션 뎁스 | 3단계 (GNB > 2depth > 3depth) |
| 주요 페이지 유형 | 목록, 등록/수정, 상세, 대시보드 |

### 1-2. 강점 (살릴 것)
- **정보 밀도**: 목록 페이지에 많은 데이터를 한눈에 보여주는 구조 → 유지
- **3단계 메뉴**: 기능이 체계적으로 분류되어 있음 → 개선하여 유지
- **iframe 기반 독립 콘텐츠**: 페이지 이동 시 레이아웃 유지 → 유지
- **다크 사이드바**: 이미 구현된 방향성 → 고도화

### 1-3. 문제점 (개선할 것)
- 57개 모듈을 스크롤 없이 탐색하기 어려움 → **퀵서치 + 즐겨찾기**
- 목록 페이지의 검색/필터 영역이 콘텐츠보다 큰 비중 → **검색 영역 컴팩트화**
- 테이블이 많은 컬럼을 나열할 때 가독성 저하 → **테이블 UX 개선**
- 폼 페이지의 섹션 구분 없이 긴 입력 나열 → **섹션 탭/아코디언 분리**
- 상태(승인대기, 미결제 등) 파악을 위해 직접 진입해야 함 → **대시보드 집중화**

> **"Dense but Calm"** — 많은 데이터를 다루지만 피로하지 않은 관리자 화면

- 데이터 밀도는 최대한 유지하면서 **시각적 계층**으로 숨 쉬는 공간 확보
- 2026년 SaaS 어드민 트렌드: Vercel Dashboard, Linear, Retool 참조
- 핵심 원칙: **보기 → 찾기 → 처리** 3단계 흐름을 빠르게

## 3. 레이아웃 구조

### 3-1. 전체 레이아웃 (변경 없음, 고도화)
```
┌─────────────────────────────────────────────┐
│  사이드바 (250px)  │  콘텐츠 영역 (flex:1)    │
│                    │  ┌───────────────────┐  │
│  [로고]            │  │    탑바 (50px)     │  │
│  ─────────────     │  └───────────────────┘  │
│  [퀵서치]          │  ┌───────────────────┐  │
│                    │  │                   │  │
│  [즐겨찾기]        │  │   iframe 콘텐츠   │  │
│                    │  │                   │  │
│  ─────────────     │  │   (목록/폼/대시)  │  │
│  [1depth 메뉴]     │  │                   │  │
│    ├ [2depth]      │  │                   │  │
│    │   └ [3depth]  │  └───────────────────┘  │
│    └ [2depth]      │                         │
└─────────────────────────────────────────────┘
```

### 3-2. 사이드바 개선사항
| 요소 | 현재 | 개선 |
|------|------|------|
| 퀵서치 | 없음 | 사이드바 상단에 `Ctrl+K` 단축키 서치박스 추가 |
| 즐겨찾기 | 없음 | 자주 쓰는 메뉴 핀 고정 (세션 저장) |
| 메뉴 뱃지 | 없음 | 처리 대기 건수 뱃지 표시 (예: 결제대기 3) |
| 아이콘 | 없음 | 각 1depth에 아이콘 추가 (Material Symbols 사용) |
| 접기 | 없음 | 사이드바 접기 → 아이콘만 보이는 60px 모드 |
| 활성 상태 | 파란 텍스트 | 좌측 3px 인디케이터 + 약한 배경색 |

### 3-3. 탑바 개선사항
| 요소 | 현재 | 개선 |
|------|------|------|
| 브레드크럼 | 없음 | 현재 위치 경로 표시 (예: 회원관리 > 회원목록) |
| 빠른 작업 | 없음 | 새 항목 추가 빠른 버튼 (+) |
| 알림 | 없음 | 알림 아이콘 + 드롭다운 (처리 대기 항목) |
| 사이트 전환 | 별도 영역 | 탑바 우측으로 이동 (다중 사이트 관리용) |

## 4. 페이지 유형별 개선

### 4-1. 목록 페이지 (가장 많이 사용)

#### 현재 구조
```
[검색 영역 — 큰 박스] [조회 버튼]
[도구 버튼들]
[테이블 헤더]
[테이블 행 x N]
[페이징]
```

#### 개선 구조
```
┌─ 페이지 헤더 ─────────────────────────────────────┐
│  회원 목록                    [엑셀] [+ 신규등록]  │
│  전체 1,234명 · 오늘 가입 12명                     │
└────────────────────────────────────────────────────┘
┌─ 검색 필터 (기본 접힘, 클릭 시 펼침) ─────────────┐
│  [검색어___________] [기간▼] [상태▼] [조회]  [▾]  │
│  (펼침 시) 상세 검색 옵션 표시                     │
└────────────────────────────────────────────────────┘
┌─ 테이블 ───────────────────────────────────────────┐
│  □  번호  이름    연락처  가입일    상태   관리     │
│  ── 헤더 배경 #f8fafc, 12px 볼드, border-bottom ─  │
│  □   1  홍길동  010-...  2026-01  [정상]  [수정]   │
│  □   2  김철수  010-...  2026-01  [대기]  [수정]   │
│  ─ 행 hover: #f8fafc, 체크시 #eff6ff ─────────────  │
│  [선택 0건] [일괄삭제] [일괄승인]  ← 선택 시 등장  │
└────────────────────────────────────────────────────┘
│  < 1 2 3 ... 10 >   10개씩▼   총 1,234건           │
```

#### 핵심 개선 포인트
1. **검색 영역 컴팩트화**: 기본적으로 한 줄 필터 (키워드 + 주요 필터 2~3개), 상세 검색은 토글
2. **페이지 헤더 메타정보**: 전체 건수, 오늘 증가분 등 컨텍스트 정보 제공
3. **일괄 처리 액션바**: 체크박스 선택 시에만 등장하는 플로팅 액션바
4. **상태 배지 통일**: 모든 목록의 상태 컬럼을 일관된 배지 스타일로 통일
5. **인라인 주요 액션**: 수정 링크 대신 아이콘 버튼 (연필, 삭제, 복사)
6. **컬럼 고정**: 번호, 이름(첫 컬럼)과 관리(마지막 컬럼) 고정, 중간 스크롤
7. **테이블 헤더 고정**: 세로 스크롤 시 헤더 고정 (`position: sticky`)

#### 핵심 개선 포인트
1. **탭/섹션 분리**: 기본정보 / 상세정보 / 설정 / 이력으로 섹션화 (탭 또는 아코디언)
2. **2컬럼 레이아웃**: 좌측 메인 입력, 우측 상태/메모/요약 사이드패널
3. **하단 스티키 액션바**: 긴 폼에서도 저장 버튼이 항상 보임
4. **인라인 유효성 검사**: 입력 완료 시 즉시 피드백 (success 테두리 + 오류 메시지)
5. **뒤로가기 컨텍스트**: 페이지 헤더에 출처 목록 링크 제공

## 6. 인터랙션 패턴

### 6-1. 퀵서치 (`Ctrl+K`)
- 사이드바 상단 또는 전체 레이아웃 최상위에 단축키 팔레트
- 입력 시 메뉴명 + 기능명 퍼지 검색
- 결과 클릭 시 해당 메뉴 활성화 + iframe 로드
- 구현: 사이드바에 검색 인풋 추가, JS로 메뉴 트리 검색

### 6-2. 즐겨찾기 (핀 고정)
- 각 메뉴 항목에 핀 아이콘 (hover 시 등장)
- 클릭 시 사이드바 상단 "즐겨찾기" 섹션에 추가
- 저장: localStorage (페이지 새로고침 후에도 유지)

### 6-3. 처리 대기 뱃지
- menu2.jsp에서 카운트 데이터 추가
- 결제대기, 환불신청, 미답변 문의 등 카운트를 메뉴 항목에 표시
- 5분 간격 AJAX 폴링으로 실시간 업데이트

### 6-4. 저장 피드백
- 저장 성공 시: 우측 하단 토스트 알림 (3초)
- 저장 실패 시: 해당 필드 인라인 오류 + 상단 요약
- 로딩 중: 저장 버튼 disabled + 스피너

### 6-5. 일괄 처리
- 체크박스 선택 시 테이블 하단에 플로팅 액션바 등장
- "N건 선택됨" + [삭제] [상태변경▼] [내보내기] 버튼
- 전체 선택 시 현재 페이지 vs 전체 데이터 선택 구분 메시지

## 9. 구현 우선순위 및 단계별 계획

### Phase 1 — 프레임/공통 (1~2주)
